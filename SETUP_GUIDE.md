# 가지 생각도구 (Branch) 프로젝트 설정 및 API 가이드

이 문서는 프로젝트 초기 생성 시 사용된 명령어와, 특히 중요했던 API 키 설정 방법 및 주의사항을 정리한 가이드입니다.

---

## 1. 초기 설치 및 설정 명령어 (Terminal History)

프로젝트를 구동하고 주요 기능을 구현하기 위해 터미널에서 실행된 핵심 명령어들입니다.

### 핵심 패키지 설치
```bash
npm install @xyflow/react lucide-react motion react-markdown @google/genai
```
*   **@xyflow/react**: 로직 트리를 그리기 위한 핵심 라이브러리 (React Flow의 최신 버전)
*   **lucide-react**: 깔끔한 인터페이스를 위한 아이콘 모음
*   **motion**: 부드러운 애니메이션 효과
*   **react-markdown**: AI의 답변을 보기 좋게 렌더링하기 위한 도구
*   **@google/genai**: Google Gemini AI 모델과 통신하기 위한 SDK

### 추가 패키지 설치 (이미지 저장 기능)
```bash
npm install html2canvas
```
*   **html2canvas**: 현재 화면의 로직 트리를 캡처하여 이미지 파일로 저장하기 위해 추가되었습니다.

## 2. Gemini API 키 설정 가이드

AI Studio 환경에서 API 키를 설정할 때 발생했던 오류들을 바탕으로 정리한 올바른 설정 방법입니다.

### ✅ 설정 단계
1.  **키 발급**: [Google AI Studio](https://aistudio.google.com/app/apikey)에서 본인의 API Key를 발급받습니다.
2.  **Secrets 메뉴 접속**: AI Studio 우측 상단의 **'Settings'**를 클릭한 후 **'Secrets'** 탭을 선택합니다.
3.  **값 입력**:
    *   **Name**: `GEMINI_API_KEY` (반드시 대문자로 정확히 입력해야 합니다)
    *   **Value**: 발급받은 키(`AIza...`로 시작하는 긴 문자열)를 입력합니다.
4.  **저장 및 선택**: 입력 후 드롭다운 메뉴에서 해당 키가 **선택(Selected)**된 상태인지 확인합니다.

### ⚠️ 중요 유의사항 및 트러블슈팅
*   **드롭다운 선택 필수**: 단순히 값을 입력하는 것뿐만 아니라, 드롭다운 메뉴에서 활성화할 키가 체크되어 있어야 앱에서 `process.env.GEMINI_API_KEY`로 읽어올 수 있습니다.
*   **서버 vs 클라이언트**: 초기에는 보안을 위해 서버 측(`server.ts`)에서 처리하려 했으나, AI Studio 프리뷰 환경의 특성상 클라이언트에서 직접 호출하되 **Secrets** 기능을 활용하는 것이 가장 안정적이었습니다.
*   **보안**: `.env` 파일이나 코드 내부에 키를 직접 적어넣는 방식은 소스 코드를 공유할 때 키가 유출될 위험이 큽니다. 반드시 **Secrets** 기능을 통해 관리하세요.
*   **유효성 오류**: "API key not valid" 메시지가 뜰 경우, 키 자체가 만료되었거나 복사 과정에서 앞뒤에 공백이나 따옴표(`"`, `'`)가 포함되지 않았는지 확인해야 합니다.

## 3. 앱 수정 이력 요약
*   **시작점(행동/상태)**: 기존 'Root'나 '뿌리'라는 표현 대신 사용자가 이해하기 쉬운 한국어 표현으로 통일했습니다.
*   **근본 원인**: 논리적 분석의 깊이를 더하기 위해 Root Cause를 '근본 원인'으로 명시했습니다.
*   **검토 단계 하이라이트**: 각 단계(인과관계/해결책) 진입 시 해당 버튼과 인디케이터가 명확히 구분되어 유지되도록 상태 관리를 개선했습니다.
